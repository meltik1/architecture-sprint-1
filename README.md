# Задание 1
Решил использовать подход с Webpack Module Federation. Все микрофронты 
написаны на одном фреймворке React

Задание выполнил в полном объеме, можно запустить сервисы: 
auth - npm start
cards - npm start
host - npm start

Заходить на фронт надо по ссылке: http://localhost:8080/



Монолитное приложение фронтенда разбил на три микрофронта:

1. Host - аггрегирующий микрофронтенд, который подтягивает компоненты остальных микрофронтов через webpack module federation 
2. Cards - микрофронтенд, отвечающий за карточки картинок. В нем хранятся компоненты, отвечающие за рендеринг списка картинок, компоненты для изменения карточек, и постановки лайков 
3. Auth - микрофронтенд, отвечаюший за компоненты, связанные с пользовтелем, регистрация, логин, изменение данных. Изначально хотел разделить регистрация-логин и профиль пользователя, но получался слишком мелкий фронтенд,  который по моему мнению сликшом нечасто подвергался бы изменению. Поэтому решил объединить эти два домена в один микросервис

Одно допущение в рамках учебного задания: все API вызовы оставил в самом приложении host, так 
как многие данные шарятся между компонентами, и не хотелось заниматься 
подкючением Redux, чтобы получать некоторые общие для нескольких компонет 
State. Поэтому решил передавать все State через props из родительского 
компонента

Таким, образом мы сможем разделить работу фронтендеров компании на две команды:
1. Первая отвечает за карточки с картинками
2. Вторая за аутентификация и авторизация пользователей + показ пользовательсих данных и их изменение 



# Задание 2

Разбиение монолитной системы на микросервисы 

## Предметная область

Система - позволяет создавать свои торговые площадки, на которых 
пользователи могут 
1. размешать товары и услуги
2. Покупать товары и услуги 
3. Устраивать аукцион для товаров и услуг 

Товары можно покупать, как на аукционе, так и через "корзину".

Жизненный цикл заказа:
1. При добавлении товара или услуги в корзину создается черновик заявки 
2. При очищении корзины заявка удаляется
3. При размещении заказа, товар резервируется, у клиента списываются деньги. В случае ошибки, применяются компенсирующие транзакции
4. Продавец подтверждает заказ. Товар списывается из системы 
5. В случае отказа продавца, резерв снимается и происходит отмена денежной транзакции.

Жизненный цикл аукциона:
1. Пользователь создает аукцион, указывает:
   1. ID товара, предварительно добавленного в систему
   2. Начальную цену 
2. Друние пользователи площадки делают ставки. 
3. После завершения аукциона размещается заказ из сервиса аукционов

## Микросервисы
Разбил систему следующим образом: 
1. API Gateway как единая точка входа. Функции: логирование запросов, проверка jwt токена. Проверяет JWT токен, чтобы не пегружать микросервис auth
2. Load balancer после API gateway для балансировки запросов до микросервисов.
3. Сервис Goods - домен товаров. Все CRUD операции и бизнес логика товаров, в том числе получения списка товаров, фильтрация и сортировка 
4. Сервис Services - домен услуг. Все CRUD операции и бизнес логика товаров, в том числе получения списка товаров, фильтрация и сортировка 
5. Сервис Order Orchestrtor -  одновременно управляет бизнес логикой заказов и является Saga оркестратором для саги оформления заказа. В случае ошибки при публикации заказа запускает компенсирующие транзакции в сервисы Payments и Goods 
6. Сервис Payments - управляет бизнес логикой платежей  и интегрируется с внешними системами оплаты. 
7. Сервис Auction - бизнес логика Аукциона. При завершении аукциона - запускает процесс размещения заказа, посылая запрос в сервис  Orders, в этом случае цена не расчитывается динамически, как в случае с оформлением заказа без аукциона, а присылается вместе с запросом от аукциона
8. Сервис Statistics - собириает статистику из других сервисов. Используется база Clickhouse для OLAP запросов 
8. Сервис Notifications - сервис отвечающий за отправку нотификаций пользователя по разным каналам уведомлениям. Собирает события из шины данных.
9. Сервис Support - бэкенд для сотрулников саппорта и админов. CRUD для заявок в сппорт.
